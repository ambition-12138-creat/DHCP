# DHCP 协议模拟系统 — 用户手册

## 访问方式

浏览器打开 `http://82.157.113.13:6080`，进入 noVNC 远程桌面，稍等片刻即可看到 DHCP 模拟器主界面。

## 界面布局

```
┌──────────────────────────────────────────────────────┐
│ 客户端 [AA:BB:CC:DD:00:01 ▼] ▶ 开始  ▶▶ 全部  ...  │  ← 工具栏
├────────────┬──────────────────────┬──────────────────┤
│ 交互流程   │   IP 地址池          │   系统日志       │
│ (可停靠)   │   ┌──────────────┐   │   (可停靠)      │
│            │   │ 租约信息      │   │                 │
│            │   └──────────────┘   │                 │
├────────────┴──────────────────────┴──────────────────┤
│  状态信息                          IP池: 3/51 (5%)   │  ← 状态栏
└──────────────────────────────────────────────────────┘
```

- **工具栏**：顶部按钮栏，包含所有操作控件和实时统计
- **交互流程**：左侧可停靠面板，展示 DHCP 四步交互的实时过程
- **IP 地址池**：中央上方表格，显示所有 IP 的分配状态
- **租约信息**：中央下方表格，显示当前活跃租约详情
- **系统日志**：右侧可停靠面板，终端风格日志输出
- **视图菜单**：菜单栏「视图」可开关左右面板

面板可拖动、关闭、浮动；关闭后通过「视图」菜单重新打开。

## 操作指南

### 选择客户端

工具栏下拉框选择目标客户端，格式为 `MAC地址  [状态]`。

### 按钮说明

| 按钮 | 功能 | 说明 |
|------|------|------|
| ▶ 开始模拟 | 对选中客户端执行 DHCP 四步交互 | Discover → Offer → Request → ACK |
| ▶▶ 全部模拟 | 对所有客户端依次执行 | 逐个触发完整的 DHCP 流程 |
| ↻ 续租 | 续租选中客户端的 IP | 客户端进入 Renewing 状态 |
| ✕ 释放 | 释放选中客户端的 IP | 归还 IP 到地址池 |
| ⟲ 重置 | 重置整个系统 | 清空所有分配和流程记录 |
| ＋ 添加 | 添加新客户端 | 输入 MAC 地址 (如 AA:BB:CC:DD:00:04) |
| － 移除 | 移除选中客户端 | 自动释放其 IP 租约 |

### IP 地址池

- **绿色行**：可用 IP，待分配
- **蓝色行**：已分配 IP，显示绑定客户端和分配时间

### 租约信息

- **剩余时间颜色**：绿色(>10分钟) → 黄色(<10分钟) → 红色(<1分钟)
- **状态颜色**：绿色=活跃，蓝色=已续租，红色=已过期，灰色=已释放

## 服务器管理

```bash
# 启动
cd /root/DhcpSimulator && docker compose up -d

# 停止
docker compose down

# 查看日志
docker compose logs -f

# 重新构建
docker compose up -d --build
```

## DHCP 协议流程

模拟器复现标准 DHCP 四步交互：

1. **Discover** — 客户端广播寻找 DHCP 服务器
2. **Offer** — 服务器响应，提供可用 IP
3. **Request** — 客户端请求使用该 IP
4. **ACK** — 服务器确认分配，绑定 IP

交互流程面板中用不同颜色和图标区分各步骤：● Discover · ◆ Offer · ▲ Request · ★ ACK · ↻ Renew
